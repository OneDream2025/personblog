# 响应式个人博客网站 - 功能迭代执行指令

---

## 🎯 任务总览

**执行模式**：功能迭代开发  
**约束级别**：严格执行，不允许偏离  

---

## 📋 项目基础规范

### 技术栈
```
HTML5 + CSS3 + ES6+ Module (纯原生, 无任何框架)
```

### 强制规范
- CSS 命名：强制 BEM 规范 `block__element--modifier`
- JS 模块化：ES6 import/export
- 所有函数必须有 JSDoc 注释
- 不允许内联样式，所有样式必须写在 CSS 文件

### 禁止事项 ❌
- ❌ 禁止引入任何第三方库、框架、CDN
- ❌ 禁止使用任何前端框架和 UI 库
- ❌ 禁止重构现有架构，只做增量开发

---

## ✨ 功能迭代开发 (共 2 个)

**执行要求**：按顺序开发，完成一个验证通过后再进行下一个

---

### 功能迭代 #1：文章点赞功能

**功能描述**：
为所有文章添加持久化点赞功能

**实现需求**：

| 模块 | 需求说明 |
|------|----------|
| **文章卡片** | 右下角显示❤️图标 + 数字，鼠标悬停有反馈 |
| **文章详情页** | 标题下方显示点赞区域，较大尺寸的爱心按钮 |
| **交互** | 点击爱心→变红+数字+1，再次点击→恢复原色+数字-1 |
| **持久化** | localStorage 存储，刷新页面数据不丢失 |
| **去重** | 同一浏览器对同一文章只能点赞一次 |

**新增/修改文件**：
```
├── css/components/like.css (新建)
├── modules/like.js (新建)
├── css/components/card.css (修改)
├── pages/detail.html (修改)
└── js/main.js (修改)
```

**验收标准**：
✅ 所有文章卡片显示点赞数
✅ 点击爱心有缩放/颜色动画效果
✅ 刷新页面点赞数保持不变
✅ 详情页和列表页点赞数实时同步
✅ 移动端样式适配正常

---

### 功能迭代 #2：阅读历史记录

**功能描述**：
自动记录用户浏览轨迹，侧边栏显示最近阅读列表

**实现需求**：

| 模块 | 需求说明 |
|------|----------|
| **自动记录** | 进入任意文章详情页时，自动记录ID和时间 |
| **去重逻辑** | 重复访问同一文章，更新时间并排到最前 |
| **容量控制** | 最多保留 10 条，超出自动删除最旧的 |
| **侧边栏展示** | 文章列表页侧边栏新增"最近阅读"板块 |
| **时间显示** | 智能显示：刚刚、5分钟前、2小时前 |
| **清空功能** | 提供"清空历史"按钮，一键清除所有记录 |

**新增/修改文件**：
```
├── css/components/history.css (新建)
├── modules/history.js (新建)
├── css/components/sidebar.css (修改)
├── pages/articles.html (修改)
└── js/main.js (修改)
```

**验收标准**：
✅ 访问任意文章，自动出现在最近阅读列表
✅ 时间显示为人性化相对时间
✅ 最多显示 10 条记录
✅ 点击清空历史按钮后列表立即清空
✅ 点击历史记录可正常跳转到文章
